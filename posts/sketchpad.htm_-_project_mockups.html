codepad.htm

drawing pad

  • converts pen motion into perfect line segments / curves / circles etc
  • "fixes" shapes according to constraints
  • inheratance - create sub pads and copy - change of template pad changes all copies in main

constraint solver

  • prolog based
  • all definitions come from constraints - little "baked in" behavior
    • ex: html output defines html elements as rectangles defined as constraints on line segments etc

outputs

takes combined pad tree + constraints and outputs whatever - image / html / ios layout system / mindmap file / etc
